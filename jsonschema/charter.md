# JSON Schema Working Group Charter (Proposed)

JSON Schema is a JSON language for describing the data structures of other JSON documents.
JSON Schema is widely referenced within the IETF and by other organizations such as the 3GPP, W3C, HR Open Standards,
OpenBanking UK, OpenAPI, ETSI, C2PA, and more. Also, JSON Schema is used in many open source projects and commercial
offerings.

However, JSON Schema is described in at least 5 Internet-Drafts but has never been published as an RFC. The goal of
this working group is to produce a stable, reference specification of JSON Schema as a
Proposed Standard. This specification will contain only features and
mechanisms in JSON Schema that are in known use. Also, the working group will ensure that changes are carefully assessed to avoid breaking known deployments. To achieve this goal, the working group will:

* Document and specify fixes to any security concerns with JSON Schema.
* Identify operational (including interoperability) considerations that need to be satisfied by the JSON Schema.

The working group will not produce any other language for describing JSON data structures.

## Liaison & Coordination

Given the wide use of JSON Schema outside the IETF, the WG will proactively establish liaisons and reach out
key organizations and bodies that rely on JSON Schema to seek for their inputs and establish an informed
view of deployed features and any other operational constraints.

## Milestones

* Send Liaisons to a set of organizations to inform them about this effort.
* Adopt an Informational document that inventories operational and security issues and known hindrances that prevented from converging on a reference specification. This document does not need to be published as an RFC.
* Request publication of JSON Schema as a Proposed Standard within one year of the formation of this working group.
